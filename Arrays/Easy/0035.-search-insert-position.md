# 35. Search Insert Position

> 🟢 Easy &nbsp;&nbsp; 🏷️ Arrays &nbsp;&nbsp;  🏷️ Binary Search &nbsp;&nbsp;

---

## Problem Description
Given a sorted array of distinct integers `nums` and a target value `target`, return the index if the target is found. If not, return the index where it would be if it were inserted in order.

You must write an algorithm with $O(\log n)$ runtime complexity.

### Example 1:
> **Input:** `nums = [1,3,5,6]`, `target = 5`
> **Output:** `2`

### Example 2:
> **Input:** `nums = [1,3,5,6]`, `target = 2`
> **Output:** `1`

### Example 3:
> **Input:** `nums = [1,3,5,6]`, `target = 7`
> **Output:** `4`

### Constraints:
* `1 <= nums.length <= 10^4`
* `-10^4 <= nums[i] <= 10^4`
* `nums` contains **distinct** values sorted in **ascending** order.
* `-10^4 <= target <= 10^4`

---

## Optimal Approach
> Use a standard binary search approach ($O(\log n)$ complexity). If the target is not found by the end of the loop, the left-bound pointer will naturally land exactly on the index position where the element should be inserted.

---

## Standard Binary Search Approach

```java
    public int searchInsert(int[] nums, int target) {
        Objects.requireNonNull(nums, "Input array cannot be null");
        
        int left = 0;
        int right = nums.length - 1;
        
        while (left <= right) {
            // Java 8 optimization: prevent integer overflow
            int mid = left + (right - left) / 2;
            
            if (nums[mid] == target) {
                return mid;
            } else if (nums[mid] < target) {
                left = mid + 1;
            } else {
                right = mid - 1;
            }
        }
        
        // If not found, left represents the correct sorted insertion index
        return left;
    }
```

## The Native API Approach (Arrays.binarySearch): 
> Java's standard library provides a highly optimized binary search out of the box. The trick with Arrays.binarySearch is handling the return value when an item is not found: it returns -(insertion point) - 1.By applying a bitwise bit-flip operator (~), we can decode the insertion index cleanly in a single step.javaimport java.util.Arrays;
```
    public int searchInsert(int[] nums, int target) {
        if (nums == null || nums.length == 0) return 0;

        int result = Arrays.binarySearch(nums, target);
        
        // If result >= 0, the target was found at that index.
        // If result < 0, ~result cleanly decodes the exact bitwise insertion position.
        return result >= 0 ? result : ~result;
    }
```
 > Use code with caution.Why it's great: Extremely clean code footprint that relies on tested, native JVM bytecode optimizations.Bitwise Alternative: ~result is identical to doing -(result + 1).
 ## Functional IntStream Binary Search Wrapper:
 > If you want to construct a pure functional pipeline using the Java 8 Streams context to find or infer index placement, you can isolate bounds checks directly inside a functional calculation block.
```
public int searchInsert(int[] nums, int target) {
        if (nums == null || nums.length == 0) return 0;

        // Use Java 8 Optional Int to handle conditional outcomes functionally
        OptionalInt preciseMatch = IntStream.range(0, nums.length)
                .filter(i -> nums[i] == target)
                .findFirst();

        // Fallback to structural deduction if exact match isn't present
        return preciseMatch.orElseGet(() -> 
            (int) Arrays.stream(nums).filter(num -> num < target).count()
        );
    }
```
> Use code with caution.Warning: While this showcases continuous Java 8 pipelines, the fallback .count() filter forces an O(n) time complexity scan. This defeats the mandatory \(O(\log n)\) requirement for massive datasets on LeetCode.
